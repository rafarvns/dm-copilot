# Comparativo de Stacks Tecnológicas

## Backend

| Linguagem/Runtime | Pontos fortes | Melhor para |
|-------------------|---------------|-------------|
| **Node.js** | I/O assíncrono, ecossistema npm, full-stack JS | APIs REST, real-time, BFF |
| **Python** | Produtividade, ML/AI, ciência de dados | APIs, automações, data pipelines |
| **Go** | Performance, binário único, concorrência nativa | Microsserviços, CLIs, alta carga |
| **Java / Kotlin** | Tipagem forte, ecossistema maduro, JVM | Sistemas enterprise, alta carga |
| **C#/.NET** | Produtividade, ecossistema Microsoft | Enterprise, Windows, Azure |
| **Rust** | Performance máxima, segurança de memória | Sistemas críticos, WebAssembly |
| **Elixir** | Tolerância a falhas, concorrência, actor model | Real-time, telecom, alta disponibilidade |

## Frontend

| Framework | Pontos fortes | Melhor para |
|-----------|---------------|-------------|
| **React** | Ecossistema enorme, flexível | SPAs complexas, times grandes |
| **Next.js** | SSR/SSG, SEO, full-stack | Sites públicos, e-commerce |
| **Vue.js** | Curva suave, progressivo | Times menores, adoção gradual |
| **Nuxt.js** | SSR para Vue | Sites públicos com Vue |
| **Angular** | Opinionado, enterprise-ready | Grandes corporações, equipes estruturadas |
| **Svelte** | Performance, sem virtual DOM | Apps leves, performance crítica |

## Bancos de Dados

| Banco | Tipo | Melhor para |
|-------|------|-------------|
| **PostgreSQL** | Relacional | Padrão para maioria dos casos |
| **MySQL** | Relacional | Legado, WordPress, e-commerce simples |
| **MongoDB** | Documento | Schema flexível, JSONs aninhados |
| **Redis** | Chave-valor / cache | Cache, sessões, pub/sub, filas |
| **Cassandra** | Wide-column | Alta escrita, dados distribuídos globalmente |
| **DynamoDB** | Chave-valor | Serverless AWS, escala automática |
| **Elasticsearch** | Busca | Full-text search, logs, analytics |
| **ClickHouse** | Colunar OLAP | Analytics, dashboards, big data |
| **TimescaleDB** | Séries temporais | Métricas, IoT, monitoramento |

## Message Brokers

| Broker | Melhor para |
|--------|-------------|
| **RabbitMQ** | Filas tradicionais, roteamento complexo |
| **Apache Kafka** | Alta throughput, event streaming, log distribuído |
| **AWS SQS/SNS** | Serverless, integração AWS |
| **Redis Streams** | Filas simples sem overhead |
| **NATS** | Microsserviços leves, alta performance |

## Cloud Providers

| Provider | Diferencial |
|----------|-------------|
| **AWS** | Maior ecossistema, serviços gerenciados maduros |
| **GCP** | BigQuery, ML/AI, Kubernetes (GKE) |
| **Azure** | Integração Microsoft, enterprise, Active Directory |
| **Vercel/Netlify** | Frontend, deploy simplificado, serverless edge |
| **Railway/Render** | Simplicidade, custo-benefício para startups |

## Observabilidade

| Componente | Ferramentas |
|------------|------------|
| **Logs** | ELK Stack, Loki + Grafana, Datadog, CloudWatch |
| **Métricas** | Prometheus + Grafana, Datadog, New Relic |
| **Traces** | Jaeger, Zipkin, AWS X-Ray, Datadog APM |
| **All-in-one** | Datadog, New Relic, Dynatrace |
